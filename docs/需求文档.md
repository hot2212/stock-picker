# 短线选股系统 — 需求文档

> 创建日期：2025-04-07  
> 项目路径：`D:\WorkBuddy-project\gupiao\`

---

## 一、项目定位

短期交易（短线）辅助决策系统，不研究财报基本面，专注**热门题材 + 资金流向 + 技术形态**，每日自动选股并追踪收益，每周自动优化参数。

---

## 二、每日时间线

| 时间 | 事件 | 说明 |
|------|------|------|
| **9:25** | 集合竞价结束，拉取竞价数据 | 竞价量、竞价金额、竞价涨幅 |
| **9:27** | 🌅 **开盘选股** | 综合分析后选出前5支 |
| **9:30~11:30** | 早盘实时监控 | 每30秒刷新一次行情 |
| **13:00~14:25** | 午后监控 | 同上 |
| **14:30** | 🌆 **尾盘选股** | 选出埋伏次日的前5支 |
| **14:30~15:00** | 尾盘实时监控 | 每30秒刷新 |
| **15:00** | 收盘数据记录 | 记录收盘价，更新胜率 |

非交易日（周末/节假日）不运行。

## 三、选股逻辑

### 3.1 开盘选股（9:27）

筛选条件：
- 竞价金额 ≥ **800万**（竞价参与度）
- 竞价涨幅 **2% ~ 6%**（方向明确但不过热）
- 竞价量比 ≥ **2.0**（竞价放量）
- 流通市值 **50亿 ~ 500亿**（有弹性）
- 所属板块排名前5（有板块效应）
- 成交额 ≥ **3亿**（流动性）
- 换手率 **3% ~ 20%**（活跃度）
- 均线多头：`收盘价 > MA5 > MA10`（趋势向上）

### 3.2 尾盘选股（14:30）

筛选条件：
- 当日涨幅 **1% ~ 5%**（有空间）
- 14:00-14:30 涨幅 > **0.5%**（尾盘有资金进场）
- 当日主力净流入 > **500万**
- 当日成交额 ≥ **3亿**
- 收盘价 > MA5 > MA10（短期向上）
- 属于当日热门板块

### 3.3 自选股分析

用户手动添加股票，系统按以下维度评分（满分10分）：

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| 题材热度 | 30% | 所属题材是否为当日TOP3热点 |
| 技术形态 | 25% | 均线多头 + 放量突破 + MACD |
| 资金流向 | 20% | 主力净流入 + 大单占比 |
| 量价配合 | 15% | 量比≥1.5 + 价涨量增 |
| 风险控制 | 10% | 非ST、非减持公告期等 |

评分对应建议：
- **8~10分** → ✅ 可买
- **6~7分** → ⏳ 等待确认
- **<6分** → ❌ 不建议

## 四、页面结构

共4个页面：

| 页面 | 核心内容 |
|------|---------|
| **📊 今日看板** | ① 今日早盘5支（实时行情）② 昨日尾盘5支（实时对比）③ 今日尾盘5支（14:30后出现）④ 热门板块TOP5 ⑤ 题材热度 |
| **⭐ 自选股分析** | 手动添加的自选股列表，每支显示评分/买卖区间/目标价/止损价/分析依据 |
| **📅 历史记录** | 近30天早盘+尾盘选股记录，含选股快照与后续走势对比 |
| **📈 准确率看板** | 胜率曲线、累计收益、分时段统计、个股收益排名 |

### 视觉风格
- 深色模式（深蓝黑底）
- A股标准红涨绿跌
- 高信息密度，表格为主
- 优先适配桌面宽屏

## 五、数据存储

| 数据 | 存储方式 | 保留期限 |
|------|---------|---------|
| 早盘选股记录 | SQLite | 30天自动清理 |
| 尾盘选股记录 | SQLite | 30天自动清理 |
| 自选股分析记录 | SQLite | 手动删除 |
| 每支股票的行情快照 | SQLite（含选股价、收盘价、次日价等） | 随选股记录清理 |

每月1日自动清理超过30天的早盘/尾盘数据。

## 六、实时数据刷新

| 数据类型 | 刷新频率 | 数据源 |
|---------|---------|--------|
| 选股5支实时行情 | 每30秒 | 腾讯财经（不封IP） |
| 昨日尾盘实时行情 | 每30秒 | 腾讯财经 |
| 自选股实时行情 | 每30秒 | 腾讯财经 |
| 板块排名 | 每5分钟 | 东方财富 |
| 题材热度 | 每30分钟 | 同花顺热点 |
| 收盘价 | 15:00一次性 | 腾讯财经 |

---

## 七、自我优化机制

| 触发时间 | 每周六或周日 10:00 |
|---------|------------------|
| 任务内容 | 分析近30天选股数据，按每个维度/参数分组统计胜率 |
| 调整对象 | 配置文件中的阈值参数（竞价金额、涨幅区间、量比等） |
| 调整方式 | 计算各参数不同取值下的胜率 → 自动选用最优值 → 更新配置 |
| 结果输出 | 生成周报，保存在页面可查看 |

**启动建议**：先固定运行两周积累数据量，之后系统自动优化。

## 八、技术架构

| 层 | 技术 | 原因 |
|----|------|------|
| 后端 | Python FastAPI | 轻量、与数据采集无缝对接 |
| 前端 | Vue3 + Tailwind CSS + ECharts | 快速开发，图表丰富 |
| 数据库 | SQLite | 零配置，单文件轻量存储 |
| 定时任务 | APScheduler | 精确到秒级调度 |
| 数据源 | 腾讯财经 + 同花顺 + 东方财富 + 通达信 | 通过 a-stock-data 直连 |
